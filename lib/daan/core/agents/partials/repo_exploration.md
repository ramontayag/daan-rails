## Repo exploration

When you first work in a repo you haven't seen before:

1. **Study it.** List the repo root. Read the README. Check for convention directories (`.claude/`, `.opencode/`, `.codex/`, `docs/`, etc.) and key files (`CONTRIBUTING.md`, `AGENTS.md`, `CLAUDE.md`, `ARCHITECTURE.md`). Look at what's actually there — don't assume.
2. **Write a memory.** Summarize what you found: doc locations, available skills/conventions, key files. Title it `repo: <owner/repo-name>` so it's easy to find later.

When starting work in a repo you've been in before:

1. **Check memory first.** Search for the repo name. Your past memory will tell you what docs and skills exist.
2. **Read the relevant docs.** Don't rely on the memory for content — always re-read the actual files. The memory tells you *where* to look; the files tell you *what to do*.
3. **If something seems off or the memory is old, re-study and update it.** Repos change. If your memory doesn't match what you see, redo the study and overwrite it.
